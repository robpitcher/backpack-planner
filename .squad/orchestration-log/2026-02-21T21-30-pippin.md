### Work Summary

**Problem:** Dark/light theme did not visually persist when navigating between pages (Dashboard → TripPlanner). Theme state in localStorage and HTML class were correct, but pages used hardcoded light-mode color classes (`bg-white`, `text-gray-*`) instead of theme-aware CSS variables.

**Root Cause:** Hardcoded tailwind colors bypass the `.dark` CSS custom properties defined in `src/index.css`, making components always appear light.

**Secondary Issue:** `src/components/ui/sonner.tsx` imported `useTheme` from the unused `next-themes` package instead of our custom `@/lib/theme` provider.
