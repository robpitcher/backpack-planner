### Architecture Decisions

- **Optimistic UI:** All mutations update store immediately, roll back on API failure for snappy UX
- **Archive semantics:** Maps to `trip_status = 'completed'` per database schema
- **Event propagation:** `e.stopPropagation()` on menu interactions to prevent card click-through
- **Toast notifications:** sonner for all CRUD success/failure feedback
- **Archive visibility:** Hidden on already-completed trips (no-op avoidance)
