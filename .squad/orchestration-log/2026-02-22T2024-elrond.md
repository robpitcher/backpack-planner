## Key Decisions Made
1. **Azure Static Web Apps (Free tier)** for frontend hosting — native SPA support, zero cost, built-in CDN, GitHub Actions integration
2. **GitHub Actions OIDC + User-Assigned Managed Identity** — no long-lived secrets, federated credential scoped to repo:dev branch
3. **Bicep IaC at subscription scope** — creates resource group, references modules for resource-level deployments
4. **No Key Vault for MVP** — frontend env vars are public; Google OAuth secrets live in Supabase, not Azure
5. **Google OAuth flows through Supabase Auth** — Supabase handles OIDC intermediation, frontend needs only VITE_SUPABASE_URL + VITE_SUPABASE_ANON_KEY
6. **SPA routing via staticwebapp.config.json** — navigationFallback rewrites unknown paths to index.html
