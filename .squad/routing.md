# Routing Rules

## Signal → Agent Mapping

| Signal | Route To | Notes |
|--------|----------|-------|
| Architecture, scope, decisions, design review | Strider | Lead handles high-level decisions |
| Feature ideas, requirements, product design, "what should we build", brainstorming | Gandalf | PM handles product thinking |
| Frontend, UI, components, styling, maps, map interactions | Pippin | Frontend dev handles all UI work |
| Backend, API, database, services, data models, auth | Gimli | Backend dev handles server-side |
| Tests, QA, quality, edge cases, coverage | Legolas | Tester handles all testing |
| Multi-domain / "team" | Strider + relevant specialists | Lead coordinates, specialists execute |
| Code review | Strider | Lead reviews all code |

## Fallback

When ambiguous, route to Strider for triage.
